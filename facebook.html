Facebook Touchpoint 2.0

Build a world class business communication framework that empowers teams across Facebook Inc to reach businesses in a scalable and unified way

OVERVIEW

Touchpoint 2.0 is the new iteration of the UX used by Tip Creators to sent out communications to businesses and advertisers through AYMT. We currently support over 300 internal teams in reaching businesses across over 150 channels/surfaces monthly to create and edit tips

Investments in new UX made based on feedback from users who need an easier to understand interface and allow for easier extensibility

TARGET AUDIENCE

Primary: SMB/SBG

Secondary: Other FB Ads internal teams

ROLE & DURATION
Product Designer(Lead)

Research, Information Architecture, UX Flow, Interaction, Visual Design, User Testing, Meetings

Team of 1 designers, 1 support designer, 4 developers, 1 TPM, 1EPM, 1 data scientist, 1 content stratgist

2019

The Problem/Chanllenge

Touchpoint platform is not flexible enough to handle the growing and changing needs of the AYMT platform.

System requires adaptation. Partner team requirements have grown, business ecosystem management has evolved.

Running into system limitations. Code is outdated, system is complex, tools are not friendly for new tip creators or AYMT engineers.

Building on top of what we have is not easy. Current tools are not easy to update.

Research
Key Strategy

Identify our key scenarios and distinct personas

Break down the core required functionality and use cases across personas

Invest in building simple flows for required functionality/use cases (that can also handle the complex use cases)

Improve quality guardrails for higher quality communications

Expand functionality to increase visibility and control of the overall business communication experience (ecosystem clash management, tip sets, orchestration/journeys, etc)

Goals

Simple: Touchpoint 2.0 UX should allow someone with minimal training to create and launch a tip successfully. Our key challenge will be balancing the advanced features required by some users with simple flows for common usage

Flexible: Touchpoint 2.0 should be flexible enough to handle the current and future use cases our AYMT communications

Extensible: All components offered by AYMT and Touchpoint should be built in an extensible way so that other teams can leverage our features instead of duplicating existing efforts

Transparent: Touchpoint 2.0 should have pre-launch transparency explaining tip setup options, expected delivery (audience size, channels selected, destination experience) and post-launch transparency showing accurate delivery/performance information

Design Thinking
UX Design
Feature screens
Home Page

For the experienced users, they will see the main overview list to check their tips status through different channels. The most important function is this page allows users create new tips by clicking the green action button.

Main Functions

Power Search & Saved Search - Users can user the smart search bar to search “tip name”, “channel name”, “owner”, “subscriber”, “launch status”, etc to get the acurate results quickly. Also, users can save/mark any search results as the new labels after they use the search filiter. It will increases the work efficiency since the overview list changes everyday, even every hours based on the tip launch status from the marketers and engineers.

Tools Family - We have so many userful tools/platforms related to the Touchpoint 2.0 tool for the marketers and engineers to use everyday. Since TP2.0 is the main tool in AYMT tool family, we allow users easily find any other tools in this homepage by clicking the “See More Tools”. We provided the most important three tools in the left side panel to make convenience for users based on the user research data results. Designed the modal window after the users click the “see more tools”. We organised different tools as four categories, which are “General Tools”, “Admin”, “Build on AYMT”, and “Help”. Also, worked on the context carefully with content stratgist to make sure all the marketers can understand the “engineer language” while using this tool. It is also one of our design problem based on the users feedback.

Tip Info Setup

For the creation pages, users need to fill out the tip infomation (three steps) first, then choose channels and fill out the channels infomation after. Summarized the tip info creation flow in three steps, which are “Tip General Info”, “Tip Setup”, and “Audience Targeting Setup” based on the user researches and testings.

Step 1 - We wanted users to fill out some very basic infomation in the first step so that we can setup the frame of this tip from backend. (ex, tip name, description, team name, etc.)

Step 2 - We wanted the marketers to setup the destination where they want to send their tips, choose the conversion event and set up the GK (gate keeper) rules carefully by using the drop down menu design.

Step 3 - We wanted the marketers have the ability to add and narrow down what kind of audience they want to send their tips to target by using the customize and personalize function design.

Main Functions

Progress Bar - Designed the progress bar to represent different steps because it’s the best way to set expectations, give an impression of activity to make users clam down. In other words, it can decrease the posibility of the mistakes. Especially in ads team, one tip mistake from marketer would make team and company loose lots of money, and cost lots of extra resources to fix the problems.

Required Fields - To put red asterisks after the field label if the field was required. Using an asterisk to mark required fields is an easy way to improve the usability of your forms. Only marking optional fields makes it difficult for people to fill out the form.

Save to the Draft - We allow users save their temperory information to the draft anytime while they are filling out different pages.

Tooltip showout - We put tooltips out under each titles as description to explain the meaning of the content, instead of using the tooltip icon or hovered over. Based on the user testing, 80% users are not touching the tooltip icon to find the explaination of different titles, they even don’t know what is that icon means, so we decided just to show the description out so that users can easily see.

Select Channel(s)
We allow users select multiple channels in this channel setup section. We support 150 channels/surfaces in AYMT, but currently we migrating 15 plus popular of channels from 1.0 to 2.0 first, then we will design other design to support the 150 channels in this page.
Channel Setup

Users have to fill out all the required infomation for each channel they selected from last step. They don’t need to fill out all the infomation of specific channel one by one, they can save to the draft if they want to finish it other time.

Main Functions

Add & Delete Channels - Users still can add more channels in this page and also they can delete channels anytime.

Tip Info Preview / Edit - In the “Channel Setup” page, we also allow users see “tip info preview” to give users an opportunity to review the tip infomation that they already filled in, and also give users opportunity to edit the tip infomation at here by clicking the “Edit Info” button.

Data Management - Specific Channel

After users finish all the channels set up, they will see this page, which is data management page to see the “rollout” data results for their specific channel.

Rollout Steps - We provided three zone levels to let users check the rollout result for the specific channel. Some channels have different zone level process, so I designed the progress bar to let users know which rollout result/level the channel in and they will see the detailed infomation in “Zone” section and “Performance Data” section.

Control Panel - Users will see all the list of channels they selected on the left control panel. We allow users add single channel or multiple channels by clicking “Add Channel(s)”, and they also can delete channels by simply click the delete icon.

Status Indicator - I use each four color dots to indicate the status of every channels so that users can easily know which status the channel in right now without type into the each section to see the details.

Reviews - Review section is only for “Notification channel” status page. If users need to see the review details, they can click link to go to the detail page/tool to get the specific data.

28 Days Performance Data - We prefer users to see the performance status and metrics of 28 days is because it is the most accurate number to track the correct data. However, I still designed a drop down menu to give users chance to choose to see other days of the metrics.

Zones - We provide the “Quality Check” and “Next Step” to the users to find the reason why the channel is in this status currently, and we also provide the workable suggestions for users without looking into the “Quality Data Details” section to figure out how to solve the problem.

Quality Data Details - In the “Zones” section, I designed a hidden extensible section for users to see the quality data details if they don’t want to follow the “suggestions” that we provide to them.

Channel Info Preview
We allow users to edit the channel info in this section. Based on the users research, most of users like to check or change the channel info when they are in the “Data Management” section, so we decided to have edit mode design of the preview section first, then users can click “Edit Info” to turn to the edit mode to change the infomation and save it in the end.
