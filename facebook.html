Facebook Touchpoint 2.0

Build a world class business communication framework that empowers teams across Facebook Inc to reach businesses in a scalable and unified way

OVERVIEW

Touchpoint 2.0 is the new iteration of the UX used by Tip Creators to sent out communications to businesses and advertisers through AYMT. We currently support over 300 internal teams in reaching businesses across over 150 channels/surfaces monthly to create and edit tips

Investments in new UX made based on feedback from users who need an easier to understand interface and allow for easier extensibility

TARGET AUDIENCE

Primary: SMB/SBG

Secondary: Other FB Ads internal teams

ROLE & DURATION
Product Designer(Lead)

Research, Information Architecture, UX Flow, Interaction, Visual Design, User Testing, Meetings

Team of 1 designers, 1 support designer, 4 developers, 1 TPM, 1EPM, 1 data scientist, 1 content stratgist

2019

The Problem/Chanllenge

Touchpoint platform is not flexible enough to handle the growing and changing needs of the AYMT platform.

System requires adaptation. Partner team requirements have grown, business ecosystem management has evolved.

Running into system limitations. Code is outdated, system is complex, tools are not friendly for new tip creators or AYMT engineers.

Building on top of what we have is not easy. Current tools are not easy to update.

Research
Key Strategy

Identify our key scenarios and distinct personas

Break down the core required functionality and use cases across personas

Invest in building simple flows for required functionality/use cases (that can also handle the complex use cases)

Improve quality guardrails for higher quality communications

Expand functionality to increase visibility and control of the overall business communication experience (ecosystem clash management, tip sets, orchestration/journeys, etc)

Goals

Simple: Touchpoint 2.0 UX should allow someone with minimal training to create and launch a tip successfully. Our key challenge will be balancing the advanced features required by some users with simple flows for common usage

Flexible: Touchpoint 2.0 should be flexible enough to handle the current and future use cases our AYMT communications

Extensible: All components offered by AYMT and Touchpoint should be built in an extensible way so that other teams can leverage our features instead of duplicating existing efforts

Transparent: Touchpoint 2.0 should have pre-launch transparency explaining tip setup options, expected delivery (audience size, channels selected, destination experience) and post-launch transparency showing accurate delivery/performance information

Design Thinking
UX Design
Feature screens
Home Page

For the experienced users, they will see the main overview list to check their tips status through different channels. The most important function is this page allows users create new tips by clicking the green action button.

Main Functions

Power Search & Saved Search - Users can user the smart search bar to search “tip name”, “channel name”, “owner”, “subscriber”, “launch status”, etc to get the acurate results quickly. Also, users can save/mark any search results as the new labels after they use the search filiter. It will increases the work efficiency since the overview list changes everyday, even every hours based on the tip launch status from the marketers and engineers.

Tools Family - We have so many userful tools/platforms related to the Touchpoint 2.0 tool for the marketers and engineers to use everyday. Since TP2.0 is the main tool in AYMT tool family, we allow users easily find any other tools in this homepage by clicking the “See More Tools”. We provided the most important three tools in the left side panel to make convenience for users based on the user research data results. Designed the modal window after the users click the “see more tools”. We organised different tools as four categories, which are “General Tools”, “Admin”, “Build on AYMT”, and “Help”. Also, worked on the context carefully with content stratgist to make sure all the marketers can understand the “engineer language” while using this tool. It is also one of our design problem based on the users feedback.

Tip Info Setup

For the creation pages, users need to fill out the tip infomation (three steps) first, then choose channels and fill out the channels infomation after. Summarized the tip info creation flow in three steps, which are “Tip General Info”, “Tip Setup”, and “Audience Targeting Setup” based on the user researches and testings.

Step 1 - We wanted users to fill out some very basic infomation in the first step so that we can setup the frame of this tip from backend. (ex, tip name, description, team name, etc.)

Step 2 - We wanted the marketers to setup the destination where they want to send their tips, choose the conversion event and set up the GK (gate keeper) rules carefully by using the drop down menu design.
